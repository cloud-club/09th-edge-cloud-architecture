| 영역           | AWS 서비스                                 | 사용 목적                                                                                                                                                                                                                                         |
| ------------ | --------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| DNS / 진입점    | **Amazon Route 53**                     | 관리자 대시보드, API 엔드포인트, 외부 연동 도메인의 DNS 라우팅 담당. 장애 상황에서는 헬스 체크 기반 라우팅이나 다중 리전 전환 구조로 확장 가능. Route 53은 DNS, 도메인 등록, 헬스 체크 기능을 제공하는 서비스입니다. ([AWS Documentation][1])                                                                                |
| 보안 진입점       | **AWS WAF**                             | Cloud API, Dashboard, 외부 연동 API 앞단에서 비정상 HTTP 요청, 봇성 요청, 알려진 웹 공격 패턴을 필터링. AWS WAF는 API Gateway, ALB, CloudFront 같은 리소스 앞에서 HTTP(S) 요청을 모니터링하고 보호할 수 있습니다. ([AWS Documentation][2])                                                           |
| L7 라우팅       | **Application Load Balancer**           | EKS 내부의 Mission API, Dashboard, Integration API 등으로 HTTP/HTTPS 트래픽을 분산. ALB는 클라이언트 요청을 EC2, 컨테이너, IP 등 대상 그룹으로 라우팅하고 헬스 체크 기반으로 정상 대상에만 전달합니다. ([AWS Documentation][3])                                                                       |
| API 진입점      | **Amazon API Gateway**                  | Edge Site, WMS/MES/ERP, 관리자 화면이 호출하는 Cloud API의 공통 진입점. Mission 생성, 상태 조회, 정책 배포 API를 외부에 안전하게 노출. API Gateway는 API를 생성, 게시, 유지관리, 모니터링, 보호하는 관리형 서비스입니다. ([AWS Documentation][4])                                                            |
| 애플리케이션 실행    | **Amazon EKS**                          | Global Fleet Management, Mission API, Policy/Config Service, Dashboard, ERP/MES/WMS Integration, Digital Twin API 같은 Cloud 애플리케이션을 컨테이너 기반으로 실행. EKS는 Kubernetes 클러스터를 직접 설치·운영하지 않고 사용할 수 있는 관리형 Kubernetes 서비스입니다. ([AWS Documentation][5]) |
| 이벤트 스트리밍     | **Amazon MSK**                          | Edge에서 올라오는 AMR/AGV 텔레메트리, 작업 이벤트, 장애 이벤트, 설비 이벤트를 Kafka 기반 스트림으로 수집·버퍼링·재처리. MSK는 Apache Kafka 기반 스트리밍 애플리케이션을 쉽게 구축하고 운영하게 해주는 관리형 서비스입니다. ([AWS Documentation][6])                                                                         |
| IoT 메시징      | **AWS IoT Core**                        | Edge Gateway 또는 IoT Agent가 MQTT 기반으로 Cloud와 통신하는 진입점. 경량 텔레메트리, 상태 이벤트, 정책 배포, 장치 인증 흐름에 사용. AWS IoT Rules는 MQTT topic stream을 기반으로 AWS 서비스와 연동할 수 있습니다. ([AWS Documentation][7])                                                             |
| 산업 데이터 모델링   | **AWS IoT SiteWise**                    | PLC, 설비, 생산 라인 상태 데이터를 산업 설비 단위로 모델링하고 수집·분석·시각화. 설비별 `Starving`, `Blocked`, `Fault`, Cycle Time 같은 데이터를 구조화하는 데 적합. SiteWise는 산업 장비 데이터를 대규모로 수집, 모델링, 분석, 시각화하는 관리형 서비스입니다. ([AWS Documentation][8])                                        |
| 이벤트 라우팅      | **Amazon EventBridge**                  | “작업 완료”, “장애 발생”, “정책 변경”, “정비 필요” 같은 비즈니스 이벤트를 다른 서비스로 라우팅. 예를 들어 장애 이벤트 발생 시 알림, 티켓 생성, 분석 파이프라인 실행으로 연결. EventBridge는 이벤트로 애플리케이션 컴포넌트를 연결하는 서버리스 이벤트 버스입니다. ([AWS Documentation][9])                                                      |
| 트랜잭션 DB      | **Amazon Aurora / Amazon RDS**          | Mission, Site, Fleet, Policy, 사용자, 권한, 작업 상태 같은 정합성이 중요한 운영 데이터를 저장. Aurora는 MySQL/PostgreSQL 호환 완전관리형 관계형 DB 엔진이고, RDS는 관계형 DB 설정·운영·확장을 쉽게 해주는 서비스입니다. ([AWS Documentation][10])                                                            |
| 장기 데이터 저장    | **Amazon S3**                           | AMR/AGV 운행 이력, 장애 로그, 센서 요약 데이터, 분석용 원천 데이터, Incident Evidence, Digital Twin 학습 데이터를 Data Lake 형태로 저장. S3는 데이터 레이크, 클라우드 네이티브 앱, 모바일 앱 등 다양한 용도로 데이터를 저장·관리·분석·보호하는 객체 스토리지입니다. ([AWS Documentation][11])                                     |
| 시계열 데이터      | **Amazon Timestream for LiveAnalytics** | 차량 위치, 배터리, 속도, 설비 상태, 구간 혼잡도, 처리량 같은 시간 기반 메트릭 저장. 대시보드나 이상 징후 분석에 사용. Timestream은 애플리케이션 성능, IoT 센서, 제조 설비 데이터처럼 시간에 따라 변하는 데이터를 분석하는 데 사용됩니다. ([AWS Documentation][12])                                                                    |
| 관측성          | **Amazon CloudWatch**                   | EKS 서비스, API Gateway, ALB, MSK, RDS, Edge 연동 API의 로그·메트릭·알람 수집. 운영자는 CloudWatch 기반으로 API 지연, 이벤트 적체, 장애 이벤트 증가, 서비스 이상을 확인. CloudWatch는 AWS 리소스와 애플리케이션을 실시간 모니터링하고 관측성을 제공합니다. ([AWS Documentation][13])                                     |
| 인증 / 권한      | **AWS IAM / OIDC**                      | EKS Pod, 운영자, CI/CD, 외부 시스템, Edge 연동 컴포넌트의 권한 제어. 서비스 간 최소 권한 원칙 적용. IAM은 AWS 서비스와 리소스 접근을 안전하게 제어하는 서비스입니다. ([AWS Documentation][14])                                                                                                        |
| 비밀정보 관리      | **AWS Secrets Manager**                 | DB 비밀번호, API Key, 외부 MES/WMS 연동 토큰, Edge 인증 자격 증명을 안전하게 저장·조회·교체. Secrets Manager는 시크릿 생성, 관리, 조회, 자동 교체를 지원합니다. ([AWS Documentation][15])                                                                                                    |
| 컨테이너 이미지 저장소 | **Amazon ECR**                          | Mission API, Global Fleet, Dashboard, Integration Service, Digital Twin Service 등의 컨테이너 이미지를 저장하고 EKS에서 배포할 때 사용. ECR은 안전하고 확장 가능한 AWS 관리형 컨테이너 이미지 레지스트리입니다. ([AWS Documentation][16])                                                       |
